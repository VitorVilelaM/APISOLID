SOLID é um conjunto de 5 principios

Single responsabilty principle
Cada arquivo deve ser responsável por apenas uma tarefa dentro da nossa aplicação

Open closed principle
Uma classe deve ser aberta para extensoes, mas nao para modivificações

Liskov substitution principle
Possa substituir algumas funções que uma classe depende

Interface agregation principle
Nao crie interfaces muito robustas, criando mini interfaces com menos informações

Dependency inversion principle
As classes da aplicação nao devem depender de implementações e sim de interfaces
